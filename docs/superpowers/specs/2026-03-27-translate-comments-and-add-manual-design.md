# 注释中文化与使用手册编写 - 设计规范

## 项目背景

这是一个 Android SVN 代码审查工具（SuperPower Code Review），通过结合静态代码分析和 AI 辅助审查，帮助开发团队提高 Android 项目的代码质量。

## 任务目标

1. **注释翻译**：将项目所有 Python 源代码中的英文注释全部改为中文注释
2. **编写使用手册**：在项目根目录生成一份完整的中文使用手册 `README.md`，教授他人如何使用该工具

## 范围界定

### 需要处理的文件

所有 `.py` 源代码文件：
- `src/` 目录下所有模块（主模块、AI 审查模块、报告生成模块、本地规则模块）
- 根目录下的测试文件
- **排除**：`.git/`, `__pycache__/`, `code-review-output/` 等非源代码目录

### 不需要处理的内容

- 代码逻辑和变量命名保持不变
- 文件名保持不变
- 只修改注释内容，不改变代码功能

## 设计方案

### 翻译策略

采用逐文件精翻策略：
1. 逐个读取源代码文件
2. 按行解析，识别英文注释
3. 将注释准确翻译为中文
4. 保持原有代码缩进、空行、格式完全不变
5. 写回原文件

**理由**：
- 逐文件处理风险可控，不会批量破坏代码
- 翻译质量更有保证
- 容易发现和处理异常情况

### 使用手册结构

根目录 `README.md` 将包含以下章节：

1. **项目介绍** - 什么是这个工具，解决什么问题
2. **功能特性** - 列出核心功能和支持的审查规则
3. **环境要求** - 需要的 Python 版本和依赖
4. **安装步骤** - 如何克隆安装、配置依赖
5. **配置说明** -
   - 环境变量配置（AI API Key）
   - 配置文件说明
6. **使用方法** -
   - 命令行参数说明
   - 基本使用示例
   - SVN 集成方式
7. **审查规则说明** - 每条规则检查什么内容
8. **输出报告** - 支持的报告格式说明
9. **扩展开发** - 如何添加自定义规则
10. **常见问题** - FAQ

## 质量保证

- 翻译后必须保持代码可正常运行
- 注释翻译要符合中文技术文档规范
- 保持原有代码结构和格式不变
- 翻译要准确反映原注释含义

## 实现顺序

1. 读取每个源代码文件
2. 翻译文件中所有注释
3. 保存文件并验证语法正确
4. 全部完成后生成 README.md 使用手册
